一：tomcat部署考试保障前端examvue项目
（由于是分离部署前端，为了保证端口一致，将前端examvue项目与后端exam项目部署在一块）dzkj000admin
部署参考：https://blog.csdn.net/qq_41223443/article/details/107314740
部署vue后刷新出现404参考：https://blog.csdn.net/qq_45740349/article/details/120809288

1.停止tomcat
  linux上指令：sudo systemctl stop file.java-server.service（注：sudo是强制执行，在登录账户没有操作权限下使用）
2.更换打包好的前端examvue项目
  2.1进入服务器上路径/usr/local/file/java/apache-tomcat-9.0.43/webapps/exam
  （没有自己创建，保证跟vue打包时vue.config里面设置的公共路径一致，代码如下：base: '/examvue/'）
  2.2替换掉static文件夹、favicon.ico文件、index.html文件
3.启动tomcat
  linux上指令：sudo systemctl start file.java-server.service

二：nginx上部署考试保障前端examvue项目

【前提：nginx压缩包安装参考：（为主）https://blog.csdn.net/Dreamhai/article/details/112037017，
         https://blog.csdn.net/weixin_40484847/article/details/121467520                      】

  （二级目录部署参考：https://www.jianshu.com/p/5090d2519c87）
1.进入/usr/local/file/java/web_space/examvue/nginx.conf文件添加（已做修改就可以不用变了）
  location /examvue {
            alias /usr/local/file/java/web_space/examvue/dist; # 代理二级目录 需要使用关键字 alias
            index index.html index.htm;
            # 解决页面刷新404问题
            try_files $uri $uri/ /examvue/index.html;
            #include /usr/local/nginx/conf/mime.types;
            include   mime.types;
            default_type  application/octet-stream;
        }
  
2.vue项目位置：/usr/local/file/java/web_space/examvue/
/usr/local/file/java/apache-tomcat-9.0.43/webapps/examvue/

/usr/local/file/java/apache-tomcat-9.0.43/webapps/areavue/
  直接替换dist文件即可
  
3.重启nginx
  linux中输入：sudo nginx -s reload
